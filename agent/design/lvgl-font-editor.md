# LVGL Font Editor — Design Document

## 1. Purpose

A browser-based editor for LVGL font `.c` files generated by `lv_font_conv`. Lets users visually inspect and modify 1bpp glyph bitmaps, adjust offsets, and save back to valid, compiling C code. See `agent/project.md` for full feature list.

---

## 2. Data Model (`src/model/font-model.ts`)

### FontData
Top-level container for a parsed font file.

| Field | Type | Description |
|---|---|---|
| `name` | `string` | Font variable name (e.g. `"tiny_mono_3x5"`) |
| `size` | `number` | Font size in px from header comment |
| `bpp` | `number` | Bits per pixel (always 1 for initial scope) |
| `lineHeight` | `number` | Maximum line height |
| `baseLine` | `number` | Baseline from bottom |
| `glyphs` | `Glyph[]` | Ordered array of parsed glyphs |
| `cmaps` | `CmapRange[]` | Character mapping table |
| `headerComment` | `string` | Original header comment text (preserved verbatim) |
| `fallbackFont` | `string \| null` | Fallback font reference (e.g. `"LV_FONT_UNSCII_8"`) |
| `opts` | `string` | Original `--opts` string from header comment |

### Glyph
One character's bitmap and layout data.

| Field | Type | Description |
|---|---|---|
| `unicode` | `number` | Unicode code point |
| `char` | `string` | The character (for display) |
| `advanceWidth` | `number` | Advance width (raw value from `adv_w`) |
| `boxWidth` | `number` | Bounding box width in pixels |
| `boxHeight` | `number` | Bounding box height in pixels |
| `offsetX` | `number` | X offset |
| `offsetY` | `number` | Y offset |
| `bitmap` | `boolean[][]` | 2D pixel grid, `[row][col]`, `true` = pixel on |

### CmapRange
One entry in the character mapping table.

| Field | Type | Description |
|---|---|---|
| `rangeStart` | `number` | Starting Unicode code point |
| `rangeLength` | `number` | Number of code points in range |
| `glyphIdStart` | `number` | Starting glyph ID |
| `type` | `CmapType` | Enum: `FORMAT0_TINY`, `FORMAT0_FULL`, `SPARSE_TINY` |
| `unicodeList` | `number[] \| null` | For sparse maps |
| `glyphIdOfsList` | `number[] \| null` | For FORMAT0_FULL maps |

### CmapType (enum)
`FORMAT0_TINY` | `FORMAT0_FULL` | `SPARSE_TINY`

## 3. Parser (`src/parser/lvgl-parser.ts`)

### Purpose
Reads an LVGL font `.c` file (as a string) and produces a `FontData` object.

### Public API
- `parseLvglFont(source: string): FontData` — main entry point. Throws `ParseError` on invalid input.

### Parsing Strategy
The parser uses regex-based extraction over C source text. It does **not** attempt to be a general C parser.

**Sections parsed (in order):**
1. **Header comment** — extract `size`, `bpp`, `opts` from `/* Size: N px */` etc.
2. **Font guard** — extract font name from `#if FONT_NAME`.
3. **Glyph bitmap array** — for each `/* U+XXXX "c" */` comment, capture the hex byte literals that follow until the next comment or `};`.
4. **Glyph descriptor array** — parse each `{.bitmap_index = N, .adv_w = N, .box_w = N, .box_h = N, .ofs_x = N, .ofs_y = N}` struct literal.
5. **Character map** — parse `cmaps[]` array entries.
6. **Font descriptor** — extract `bpp`, `cmap_num`.
7. **Public font** — extract `line_height`, `base_line`, font variable name, `fallback`.

**Bitmap decoding:**
Given the raw bytes for a glyph and its `box_w`/`box_h`, unpack bits MSB-first into a `boolean[][]`. Each row is `box_w` bits; rows are packed with byte-alignment per row (stride = ceil(box_w / 8) bytes per row).

### Error Handling
- `ParseError` class extends `Error` with a `section` field indicating which parsing phase failed.
- Validate: glyph count matches descriptor count; bitmap byte count matches expected size from box dimensions.

## 4. Serializer (`src/serializer/lvgl-serializer.ts`)

### Purpose
Converts a `FontData` object back into a valid LVGL font `.c` file string.

### Public API
- `serializeLvglFont(font: FontData): string` — returns the complete C file content.

### Serialization Strategy
Generates C source text section by section, matching the format of `lv_font_conv` output:
1. Header comment (preserved or regenerated from `opts`).
2. Include guards and LVGL includes (template).
3. Font guard macro.
4. `glyph_bitmap[]` — encode each glyph's `boolean[][]` into packed bytes (MSB-first, row-aligned). Emit hex literals with Unicode comments.
5. `glyph_dsc[]` — emit struct literals with computed `bitmap_index` values.
6. Character map arrays and `cmaps[]`.
7. Font descriptor.
8. Public font variable.
9. Closing `#endif`.

**Bitmap encoding** is the inverse of parsing: for each row, pack `box_w` bits MSB-first into `ceil(box_w / 8)` bytes.

### Behavior Contracts
- `parseLvglFont(serializeLvglFont(parseLvglFont(source)))` must produce a `FontData` identical to `parseLvglFont(source)` (round-trip stability).
- Output must compile as valid C when included in an LVGL project.

## 5. History / Undo-Redo (`src/history/history.ts`)

### Purpose
Provides undo/redo for all editing operations on `FontData`.

### Data Structure
Uses a snapshot-based approach: stores a stack of serialized glyph states.

| Field | Type | Description |
|---|---|---|
| `undoStack` | `Snapshot[]` | Previous states |
| `redoStack` | `Snapshot[]` | States undone |
| `maxSize` | `number` | Maximum stack depth (default: 100) |

A `Snapshot` captures the minimal state needed to restore: the glyph index, and a deep clone of the `Glyph` object.

### Public API
- `pushState(glyphIndex: number, glyph: Glyph): void` — save state before mutation. Clears redo stack.
- `undo(): Snapshot | null` — pop from undo stack, push current state to redo stack.
- `redo(): Snapshot | null` — pop from redo stack, push current state to undo stack.
- `canUndo(): boolean`
- `canRedo(): boolean`
- `clear(): void`

### Behavior Contracts
- After N push+undo calls, the original state is restored.
- Push clears the redo stack.
- Stack respects `maxSize`; oldest entries are discarded.

## 6. Editor (`src/editor/`)

### Canvas Renderer (`canvas-renderer.ts`)

Draws a glyph's `boolean[][]` onto an HTML `<canvas>` element at a configurable pixel scale.

**Public API:**
- `CanvasRenderer` class
  - `constructor(canvas: HTMLCanvasElement, pixelSize: number)`
  - `render(glyph: Glyph): void` — clears canvas and draws the glyph grid.
  - `setPixelSize(size: number): void`
  - `coordsToPixel(clientX: number, clientY: number): {row: number, col: number} | null` — converts mouse coords to glyph pixel coords.

**Rendering details:**
- Each glyph pixel is a square of `pixelSize × pixelSize` screen pixels.
- Grid lines drawn at 1px between pixels.
- `true` pixels drawn in foreground color; `false` pixels in background color.
- Canvas is sized to `(boxWidth * (pixelSize + 1)) + 1` by `(boxHeight * (pixelSize + 1)) + 1`.

### Editing Tools (`tools.ts`)

Stateless functions that mutate a `Glyph`'s bitmap.

**Public API:**
- `togglePixel(glyph: Glyph, row: number, col: number): void`
- `setPixel(glyph: Glyph, row: number, col: number, value: boolean): void`
- `moveOffset(glyph: Glyph, dx: number, dy: number): void` — adjusts `offsetX`/`offsetY`.
- `resizeFont(font: FontData, newBoxWidth: number, newBoxHeight: number): void` — resizes all glyphs, centering existing bitmap data.
- `boxSelect(glyph: Glyph, r1: number, c1: number, r2: number, c2: number): boolean[][]` — extracts a rectangular region.
- `pastePixels(glyph: Glyph, pixels: boolean[][], startRow: number, startCol: number): void` — writes a rectangular region.
- `movePixels(glyph: Glyph, selection: {r1: number, c1: number, r2: number, c2: number}, dr: number, dc: number): void` — moves selected pixels.

### Clipboard (`clipboard.ts`)

**Public API:**
- `ClipboardStore` class
  - `copy(pixels: boolean[][]): void`
  - `paste(): boolean[][] | null`
  - `hasContent(): boolean`

## 7. UI Layer (`src/ui/`)

### File I/O (`file-io.ts`)
- `openFile(): Promise<string>` — uses `<input type="file">` to read a `.c` file as text.
- `saveFile(content: string, filename: string): void` — uses `Blob` + download link to save.

### Glyph List (`glyph-list.ts`)
- `GlyphList` class
  - `constructor(container: HTMLElement, onSelect: (index: number) => void)`
  - `render(glyphs: Glyph[]): void` — populates the list with glyph entries showing Unicode, character, and a mini preview.
  - `setSelected(index: number): void`

### Toolbar (`toolbar.ts`)
- `Toolbar` class
  - `constructor(container: HTMLElement, handlers: ToolbarHandlers)`
  - Renders buttons: Open, Save, Undo, Redo, tool mode selectors (draw, select, move).
  - Updates enabled/disabled state based on context.

### ToolbarHandlers (interface)
Callbacks: `onOpen`, `onSave`, `onUndo`, `onRedo`, `onToolChange(tool: ToolMode)`.

### ToolMode (enum)
`DRAW` | `SELECT` | `MOVE`

---

## 8. Application Entry Point (`src/index.ts`)

Wires all modules together:
1. Creates DOM elements (toolbar, glyph list, canvas).
2. On file open: parse → populate glyph list → render first glyph.
3. On canvas interaction: delegate to current tool → push history → re-render.
4. On save: serialize → download.
5. Keyboard shortcuts: Ctrl+Z undo, Ctrl+Y/Ctrl+Shift+Z redo, Ctrl+C copy, Ctrl+V paste.

## 9. Error Handling Summary

| Layer | Error Type | Behavior |
|---|---|---|
| Parser | `ParseError` | Show user-facing message with section name |
| Serializer | None expected | Internal invariants — throw if model is inconsistent |
| Editor | Out-of-bounds coords | Silently ignore (no-op) |
| File I/O | File read failure | Show alert with error message |
| History | Empty stack | `undo()`/`redo()` return `null`; caller checks |
