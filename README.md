# LVGL Font Editor

A browser-based visual editor for LVGL font `.c` files generated by [lv_font_conv](https://github.com/lvgl/lv_font_conv). When converting fonts for small monochrome displays, rendering artifacts become prominent. Since the font data lives in a C source file rather than a bitmap, manual pixel-level fixes are impractical without tooling — this editor solves that.

## Features

- **Load & save** LVGL font `.c` files — output is valid, compilable C code
- **Pixel editing** — toggle individual pixels on or off with a draw tool
- **Glyph offset adjustment** — move glyph positioning within the bounding box
- **Font resizing** — change height/width for the entire font while preserving glyph offsets
- **Box select & move** — select a rectangular region of pixels and reposition it
- **Copy & paste** — copy pixel regions and paste them (Ctrl+C / Ctrl+V)
- **Undo & redo** — full history support (Ctrl+Z / Ctrl+Y)
- **1bpp support** — designed for 1 bit-per-pixel monochrome fonts

## Usage
GitHub Page: https://apolesskiy.github.io/lvgl-font-adjuster/

1. Click **Open** and select an LVGL font `.c` file (e.g. one generated by `lv_font_conv`).
2. The glyph list populates on the left — click any glyph to edit it.
3. Use the toolbar to switch between tools:
   - **Draw** — click or drag on the canvas to toggle pixels
   - **Select** — click and drag to select a rectangular region
   - **Move** — after selecting, click to move the selected pixels
4. Use **Ctrl+C** / **Ctrl+V** to copy and paste selected pixel regions.
5. Use **Undo** / **Redo** buttons or **Ctrl+Z** / **Ctrl+Y**.
6. Click **Save** to download the edited font `.c` file.


## Development

### Prerequisites

- [Node.js](https://nodejs.org/) 18+

### Install

```bash
npm install
```

### Development

```bash
npm run dev
```

Opens a local dev server (default `http://localhost:5173`). The app runs entirely in the browser.

### Build

```bash
npm run build
```

Produces a production build in `dist/`.

### Preview Production Build

```bash
npm run preview
```

### Lint

```bash
npm run lint
```

### Test

```bash
# Unit & integration tests
npm test

# End-to-end tests (requires Chromium via Playwright)
npx playwright install chromium
npm run test:e2e

# All checks (lint + test + build)
npm run check
```

## Project Structure

```
src/
  model/          # Font data types (FontData, Glyph, CmapRange)
  parser/         # LVGL .c file parser
  serializer/     # FontData → LVGL .c file serializer
  history/        # Undo/redo history
  editor/         # Canvas renderer, editing tools, clipboard
  ui/             # Toolbar, glyph list, file I/O
  index.ts        # Application entry point
tests/
  unit/           # Unit tests (Vitest)
  integration/    # Parse–serialize roundtrip tests
  e2e/            # End-to-end tests (Playwright)
```

## Technology

- **TypeScript** with strict mode
- **Vite** for bundling and dev server
- **Vitest** for unit/integration tests
- **Playwright** for E2E tests
- **Vanilla DOM + Canvas** — no UI framework

## Note
This project was a test of whether it's efficient to use AI to make one-off jigs - in this case, a UI tool to manually fix converted LVGL fonts that I needed for a small display. This was done in two prompts ($0.24 of GitHub Copilot charges for Claude Opus 4.6) and ~40min, which is less time than it would have taken me to do the edits by hand, and *way* less time than it would have taken to make a similar tool by hand.
